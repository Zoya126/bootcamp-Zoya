Disease Target Identification Notes

Key Points to Remember

  • Use precise disease names to avoid confusion (e.g., Type 2 Diabetes instead of just Diabetes).
  • Diseases may belong to hierarchical systems, with general and specific terms.
  • Classification systems like EFO (Experimental Factor Ontology) and MONDO (Monarch Disease Ontology) are critical for structured disease exploration.
  • The EBI OLS (Ontology Lookup Service) provides APIs to fetch disease data and hierarchy.
  • Understand the importance of ontology codes (e.g., EFO or MONDO) for identifying diseases.

Challenges Faced During API Fetching

  • Understanding API documentation for EBI OLS to fetch hierarchical disease data.
  • Handling API rate limits and ensuring efficient data fetching.
  • Parsing complex JSON responses to extract relevant information like disease names, descriptions, and codes.
  • Implementing search functionality with auto-suggestions and hierarchy visualization.
  • Ensuring error handling for cases like network issues or invalid search terms.

Improvements for Widget Usability

  • Provide a clean and intuitive UI for searching and exploring diseases.
  • Allow users to select diseases and copy their corresponding EFO or MONDO codes.
  • Include a brief description of selected diseases for better context.
  • Optimize API calls to minimize latency and enhance performance.
