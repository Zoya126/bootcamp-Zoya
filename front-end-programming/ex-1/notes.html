Exercise 1: Basic Front End - Notes

Important Points to Remember

Level 1: Single-File Application (HTML, CSS, JS)

  • Use a single HTML file for the structure of the page.
  • Embed CSS and JavaScript directly within the HTML file.
  • Create a table with dynamic rows and columns for student data.
  • Use JavaScript to add rows to the table dynamically with dummy data.
  • Apply a CSS class (highlight) to newly added rows to emphasize them.
  • Integrate Google Fonts to style the table, such as using sans-serif font family.
  • Display browser information (name, version, user agent, window dimensions) below the table.

Level 2: Separation of Concerns (JS in External File)

  • Move JavaScript code to an external file (script.js) to improve maintainability.
  • Link the external JavaScript file to the HTML document.
  • Load student data from a local students.json file using JavaScript.
  • Parse and render the JSON data dynamically in the table.

Level 3: Using a Library for Data Presentation

  • Use AG Grid (Community Edition) to display student data in a grid layout.
  • Load CSV data from a file (students.csv) and parse it using PapaParse.
  • Highlight rows where the grade is "A" using CSS.
  • Enable column sorting and filtering using AG Grid features.
